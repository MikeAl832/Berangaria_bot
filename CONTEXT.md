# Berangaria Memory

Домен долговременной памяти Berangaria: какие сведения о людях и группах можно сохранить и затем использовать в разговоре.

## Language

**Достоверный факт**:
Явно утверждённое сведение, сформулированное без догадок и полезное для будущего разговора. Если источник, смысл или проверка неоднозначны, это не достоверный факт и он не сохраняется.
_Avoid_: предположение, слух, вывод, наблюдение, кандидат факта

**Источник факта**:
Сам человек, о котором говорится, когда он прямо утверждает сведения о себе. Пересказ или утверждение одного участника о другом не считается достаточным источником для автоматического сохранения.
_Avoid_: рассказчик, наблюдатель, автор сообщения (если он не является субъектом факта)

**Устойчивое сведение**:
Личный факт, предпочтение, привычка или длительный проект, которые разумно использовать после текущего разговора без привязки к ближайшей дате.
_Avoid_: текущий статус, разовая деталь, краткосрочный план

**Временное состояние**:
Правдивое на момент сообщения обстоятельство или план, срок действия которого ограничен или неизвестен. Оно не становится долговременной памятью автоматически.
_Avoid_: устойчивый факт, постоянное свойство

**Автоматическое сохранение**:
Отбор устойчивых сведений из обычных сообщений без команды или другого явного разрешения пользователя. Для него действуют все ограничения на источник, ясность и достоверность.
_Avoid_: запоминание по запросу, ручная запись, принудительное сохранение

**Актуальный факт**:
Единственная непротиворечивая версия устойчивого сведения, подтверждённая последним ясным самоутверждением человека. Новая ясная версия заменяет прежнюю; двусмысленное сообщение не меняет память.
_Avoid_: последняя запись, параллельные версии, исторический факт (если он не помечен как прошлый)

**Происхождение факта (provenance)**:
Проверяемая ссылка на исходное сообщение: его автора, идентификатор, время и короткий фрагмент, из которого непосредственно следует сохранённое сведение.
_Avoid_: голый факт без источника, ссылка на весь диалог, неподтверждённый контекст

**Доказательный фрагмент**:
Буквальная непрерывная подстрока исходного сообщения, включая исходные пробелы, которая прямо утверждает сохраняемое сведение; без неё факт не допускается к автоматическому сохранению.
_Avoid_: пересказ, вывод из соседних сообщений, пояснение extractor'а

**Ясное утверждение**:
Безусловное самоутверждение человека о себе в настоящем или устойчивом времени, без модальных слов, гипотезы, цитирования или иронического переосмысления.
_Avoid_: желание, план, предположение, вопрос, цитата, сарказм

**Кандидат факта**:
Непроверенное структурированное предложение extractor'а, связанное с исходным сообщением и доказательным фрагментом. Кандидат не является памятью и не может попасть в неё без отдельного одобрения verifier'а.
_Avoid_: сохранённый факт, подтверждённое знание, черновой факт без источника

**Одобренный факт**:
Кандидат, прошедший все проверки источника, устойчивости, ясности, доказательного фрагмента и непротиворечивости. Только одобренный факт передаётся в хранилище и участвует в поиске.
_Avoid_: запись Mem0, результат extraction, факт после fail-open

**Субъект памяти**:
Человек, о котором прямо говорится в одобренном факте; в групповом чате им может быть только автор сообщения, говорящий о себе.
_Avoid_: упомянутый человек, предполагаемый субъект, вся группа

**Чувствительный факт**:
Правдивое сведение, которое нельзя автоматически хранить из-за риска вреда или утечки: секреты и credentials, точные адреса и документы, финансовые, медицинские и иные особо личные данные.
_Avoid_: любой личный факт, приватная деталь без оценки риска

**Автоматическое исправление**:
Замена устаревшего факта новым ясным самоутверждением того же человека. Специальных команд для удаления или принудительного сохранения памяти нет.
_Avoid_: ручное редактирование, команда «запомни», команда «забудь»

**Медиа-контекст**:
Описание изображения, видео, аудио или стикера, созданное vision-моделью для текущего разговора. Оно не является источником факта и не попадает в долговременную память.
_Avoid_: доказательный фрагмент, самоутверждение, медиа-факт

**Область памяти**:
Один Telegram-чат, внутри которого факт был утверждён и может использоваться. Факты из лички, группы и других групп не переносятся между областями.
_Avoid_: глобальный профиль человека, общая память между чатами, кросс-чатный факт

**Сообщение-источник**:
Одна структурированная пользовательская реплика с областью памяти, автором, Telegram message ID, временем и исходным текстом. Даже короткая реплика получает окончательное решение; поздняя правка с тем же message ID не заменяет оригинал. Каждый кандидат факта ссылается ровно на одно сообщение-источник.
_Avoid_: склеенный текстовый батч, весь тред, неразмеченная история

**Тихое сохранение**:
Автоматическое изменение памяти без отдельного сообщения пользователю; решение и причина доступны только техническому журналу.
_Avoid_: уведомление «запомнила», подтверждение каждого факта

**Очередь проверки**:
Сохраняемый в SQLite набор сообщений-источников, которые ещё не получили окончательного решения. Временный сбой проверки или хранилища оставляет сообщение в очереди и не создаёт факт.
_Avoid_: сохранённая память, оперативный буфер, fail-open запись

**Необработанное сообщение**:
Сообщение-источник, проверка которого завершилась пятью техническими сбоями. Оно не становится памятью и больше не повторяется автоматически.
_Avoid_: отклонённый факт, сохранённый факт, бесконечный retry

**Прерванное сообщение**:
Сообщение-источник, чей Telegram-ход не завершился доставкой до остановки процесса. После перезапуска оно исключается из проверки, а сырой текст стирается; это не технический retry и не память.
_Avoid_: необработанное сообщение, ожидающий источник, восстановленный ход

**Релевантный факт**:
Одобренный факт из той же области памяти, чья связь с текущим запросом проходит строгий порог поиска. Слабое совпадение не используется, даже если сам факт достоверен.
_Avoid_: любой найденный факт, низкоскоринговое совпадение, кросс-чатный результат

**Фоновая проверка**:
Асинхронная обработка очереди после доставки обычного ответа Бер. Новое сообщение не задерживается ради памяти, поэтому факт может стать доступен не сразу.
_Avoid_: проверка в критическом пути ответа, синхронное сохранение

# 📋 Шпаргалка по логированию

## 🎚️ Быстрое переключение режимов

### В `config.yaml`:

```yaml
# 🚀 Продакшн (чистая консоль)
debug: false
verbose: false
```

```yaml
# 🔬 Разработка (видны промпты)
debug: true
verbose: false
```

```yaml
# 🐛 Глубокая отладка (всё + библиотеки)
debug: true
verbose: true
```

---

## 📊 Что показывается в каждом режиме

| Элемент                          | INFO (prod)   | DEBUG (dev)  | VERBOSE   |
|----------------------------------|---------------|--------------|-----------|  
| Входящие сообщения (обрезанные)  | ✅           | ✅           | ✅       |
| Входящие сообщения (полные)      | ❌           | ✅           | ✅       |
| Токены и стоимость               | ✅           | ✅           | ✅       |
| Память: статистика               | ✅           | ✅           | ✅       |
| Память: факты (содержимое)       | ❌           | ✅           | ✅       |
| Промпты к LLM                    | ❌           | ✅           | ✅       |
| Ответы от LLM                    | ❌           | ✅           | ✅       |
| Поиск в интернете                | ✅           | ✅           | ✅       |
| Vision (статистика)              | ✅           | ✅           | ✅       |
| Vision (описание)                | ❌           | ✅           | ✅       |
| HTTP библиотеки                  | ❌           | ❌           | ✅       |
| Telegram SDK                     | ❌           | ❌           | ✅       |

---

## 📂 Файл bot.log

**ВСЕГДА содержит полный DEBUG**, независимо от режима консоли!

В Docker пишется в `/data/bot.log`, на хосте это `./bot_data/bot.log`.
Быстрый просмотр:

```bash
./logs.sh file
./logs.sh tail
./logs.sh errors
```

## 🐛 Типичные проблемы

### "Не вижу промпты в консоли"
→ Установи `debug: true` в config.yaml

### "Консоль захламлена HTTP логами"
→ Установи `verbose: false` в config.yaml

### "Нужно посмотреть старые промпты"
→ Открой `bot.log` — там ВСЕГДА полный DEBUG

### "Сообщения обрезаны"
→ Это нормально в INFO режиме. Включи DEBUG или смотри bot.log

---

## 💡 Советы

1. **Запускай в продакшне с `debug: false`** — консоль будет чистой
2. **Включай DEBUG только когда нужно** — для отладки конкретных проблем
3. **Не включай VERBOSE без надобности** — слишком много логов
4. **Смотри bot.log для разбора полётов** — там всегда полная картина
5. **Используй цвета** — они помогают быстро найти нужное

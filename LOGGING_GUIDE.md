# 📝 Гайд по логированию Berangaria Bot

## 🎯 Концепция (Вариант 3A)

Система использует **раздельные уровни логирования** для консоли и файла:

- **Консоль**: INFO (чистый вывод) или DEBUG (если `debug=true`)
- **Файл** (`bot.log`): ВСЕГДА DEBUG (полный аудит для разбора полётов)

---

## ⚙️ Режимы работы

### 1️⃣ Продакшн режим (по умолчанию)

```yaml
debug: false
verbose: false
```

**Что видно в консоли:**
- ✅ Входящие сообщения (обрезаны до 80 символов)
- ✅ Токены и стоимость API запросов
- ✅ Краткая статистика памяти (найдено X → загружено Y фактов)
- ✅ Важные события (старт, ошибки, суммаризация)
- ✅ Результаты поиска в интернете

**Что НЕ видно в консоли:**
- ❌ Полные промпты к LLM
- ❌ Полные ответы от LLM
- ❌ Детали поиска Mem0
- ❌ Содержимое загруженных фактов


## 📂 Файл логов (bot.log)

**ВСЕГДА содержит полный DEBUG**, независимо от настроек консоли.

Это значит:
- В продакшн режиме консоль чистая, а файл содержит всё для разбора
- Можно безопасно давать файл лога для диагностики, не смотря в консоль

**Формат:** Чистый текст без ANSI цветов и тегов `[color]...[/]`

В Docker файл пишется в `/data/bot.log`, который смонтирован на хост как
`./bot_data/bot.log`. По умолчанию включена ротация: 10 МБ на файл и 5
архивов. Быстрые команды:

```bash
./logs.sh          # live Docker logs
./logs.sh file     # последние строки persistent-файла
./logs.sh tail     # follow persistent-файла
./logs.sh errors   # последние WARNING/ERROR/CRITICAL
```

`full_debug_logs: true` включает подробные записи с промптами, ответами модели,
фактами памяти и vision-описаниями для файла и Dozzle. Это не включает DEBUG в
Docker-консоли; за консоль отвечает отдельный `debug: true`.

Браузерный self-hosted viewer:

```text
https://logs.titlo10.fun -> Nginx Basic Auth -> 127.0.0.1:9999 -> Dozzle
```

Контейнер `log-tail` читает `bot_data/bot.log` и отдает его в Dozzle через stdout.
Подробности: `LOG_VIEWER.md`.

## 🎨 Цветовое кодирование

### Уровни логов:
- 🔵 **DEBUG** — Cyan
- 🟢 **INFO** — Green
- 🟡 **WARNING** — Yellow
- 🔴 **ERROR** — Red
- 🔴 **CRITICAL** — Bright Red

### Инлайн теги (используются в коде):
```python
logger.info(f"Токены: [cyan]{total}[/]")          # Голубой
logger.info(f"Успех: [green]{result}[/]")         # Зелёный
logger.warning(f"[yellow]Предупреждение[/]")      # Жёлтый
logger.error(f"[red]Ошибка[/]")                   # Красный
logger.info(f"Детали: [dim]{info}[/]")            # Затемнённый (серый)
logger.info(f"[bright_green]Успех![/]")           # Яркий зелёный
```

## 🔧 Как переключать режимы

### Метод 1: config.yaml (рекомендуется)

Отредактируй `config.yaml`:

```yaml
debug: false   # INFO режим (продакшн)
verbose: false # Без библиотечных логов
```

```yaml
debug: true    # DEBUG режим (разработка)
verbose: false # Без библиотечных логов
```

```yaml
debug: true    # DEBUG режим
verbose: true  # + все библиотеки (суперподробно)
```

### Метод 2: .env файл

Можно добавить в `.env` (если поддержка добавлена):

```env
DEBUG=false
VERBOSE=false
```

## 💡 Рекомендации

### 🚀 Для продакшна:
```yaml
debug: false
verbose: false
```
- Консоль чистая, легко отследить flow
- Файл содержит всё для разбора проблем

### 🔬 Для разработки:
```yaml
debug: true
verbose: false
```
- Видишь промпты и ответы LLM
- Видишь, что загружается из памяти
- Библиотеки не мешают

### 🐛 Для глубокой отладки:
```yaml
debug: true
verbose: true
```
- Всё, включая HTTP запросы библиотек
- Используй только для диагностики конкретных проблем

## 🛠️ Техническая информация

### Глушение библиотечных логов

По умолчанию (когда `verbose=false`) глушатся:

```python
logging.getLogger("httpx").setLevel(logging.WARNING)
logging.getLogger("httpcore").setLevel(logging.WARNING)
logging.getLogger("urllib3").setLevel(logging.WARNING)
logging.getLogger("openai").setLevel(logging.WARNING)
logging.getLogger("h2").setLevel(logging.WARNING)
logging.getLogger("hpack").setLevel(logging.WARNING)
logging.getLogger("rustls").setLevel(logging.WARNING)
logging.getLogger("rquest").setLevel(logging.WARNING)
logging.getLogger("fastembed").setLevel(logging.WARNING)
logging.getLogger("telegram").setLevel(logging.INFO)
logging.getLogger("telegram.ext").setLevel(logging.INFO)
```

### Структура log_setup.py

```python
def setup_logging(level=logging.INFO, log_file="bot.log", debug=False, verbose=False):
    root = logging.getLogger()
    root.setLevel(logging.DEBUG)  # Root всегда DEBUG для файла
    
    # Консоль — INFO или DEBUG
    console = logging.StreamHandler()
    console.setLevel(logging.DEBUG if debug or verbose else logging.INFO)
    
    # Файл — ВСЕГДА DEBUG
    file_handler = logging.FileHandler(log_file, encoding="utf-8")
    file_handler.setLevel(logging.DEBUG)
```

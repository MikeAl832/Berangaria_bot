# 🎨 Гайд по улучшению качества vision-ответов

## Проблема

DeepSeek получает описания медиа от Gemini и должен на них реагировать естественно, но есть несколько точек трения:

### 1. **Языковой конфликт** 
- Системный промпт на английском
- Описания от Gemini на русском  
- DeepSeek должен переключаться между языками в одном контексте

### 2. **Слишком техническая структура описаний**
Старый формат (до исправлений):
```
1. ДЕТАЛИ: подробно перечисли всё важное...
2. УЗНАВАНИЕ: попробуй опознать...
3. ИТОГ: 2-3 предложения...
```

Проблемы:
- Звучит как отчёт аналитика, а не рассказ друга
- DeepSeek видит структуру и может невольно на неё ссылаться
- Теряется естественность — бот получает "техническое описание", а не "живое наблюдение"

### 3. **Недостаточно сильный запрет на метафразы**
DeepSeek всё равно может сказать:
- "судя по описанию"
- "на картинке видно"
- "в кадре заметно"

Потому что структура слишком явно выглядит как "пересказ чужого описания"

## Решение

### ✅ Изменения в `vision_provider.py`

#### **Новый формат промптов для Gemini**

**Для изображений:**
```python
_IMAGE_PROMPT = (
    "Опиши это изображение так, как ты рассказал бы другу в переписке. "
    "Твой ответ будет передан в текстовый чат, где тебя не видят — только читают твои слова.\n\n"
    
    "Что упомянуть обязательно:\n"
    "• Кто/что в кадре — люди (внешность, одежда, эмоции), объекты, обстановка\n"
    "• Текст и логотипы (процитируй дословно)\n"
    "• Узнаваемые персонажи, мемы, бренды — назови конкретно\n"
    "• Настроение, атмосферу\n\n"
    
    "Важно:\n"
    "- Говори естественно, без заголовков — как в сообщении мессенджера\n"
    "- Не начинай с «На картинке...» — сразу к делу\n"
    "- Если что-то смешное — можешь прокомментировать\n"
)
```

**Преимущества:**
- Звучит как инструкция для человека, который будет рассказывать
- Нет жёсткой структуры ДЕТАЛИ/УЗНАВАНИЕ/ИТОГ
- Gemini пишет более естественно → DeepSeek получает более естественный текст

#### **Для видео:**
```python
def _video_prompt(duration: float) -> str:
    return (
        f"Это видео длиной {duration:.0f} секунд. "
        "Опиши его так, как рассказал бы другу в переписке...\n\n"
        "Что упомянуть:\n"
        "• Кто/что в кадре, что делают\n"
        "• Узнаваемые персонажи/мемы\n"
        "• Как меняется сцена: начало → развитие → концовка\n"
        "• Настроение, музыка\n\n"
        "Рассказывай естественно, без заголовков..."
    )
```

### ✅ Изменения в `llm_client.py`

#### **Новый VISION_PROMPT_SUFFIX**

**Старая версия (проблемная):**
```python
"The description arrives in a structured format: 
sections «DETAILS», «RECOGNITION», «SUMMARY»"
```

**Новая версия:**
```python
VISION_PROMPT_SUFFIX = """
=== IMAGES AND VIDEO ===
When a user sends media, you receive [Image description: ...] or [Video description: ...]
These descriptions come from a vision model that saw the media and described it naturally — 
like a friend telling you what's in the picture.

The description includes:
- What's visible: people, objects, text, logos, setting
- Recognized characters/memes/brands with specific names
- Mood and atmosphere
- For video: how the scene evolves

How to use it:
✓ React naturally as if you saw it yourself
✓ Reference recognized things by name — this is your advantage
✓ If description says "похоже на..." — mention with uncertainty
✓ If it says didn't recognize — don't invent names

What NOT to do:
✗ NEVER write "visible in the picture", "judging by the description"
✗ Don't say "the description mentions..."
✗ Don't quote the description structure

Treat the description as your own observation.
"""
```

**Почему это лучше:**
- Убрал упоминание секций DETAILS/RECOGNITION/SUMMARY
- Добавил метафору "friend telling you" — снижает дистанцию
- Более сильный запрет на метафразы с примерами
- Явно сказано: "treat as your own observation"


Оцените:

### 2. **Проверьте ответы бота**
Отправьте картинку и проверьте, не использует ли бот фразы:
- ❌ "судя по описанию"
- ❌ "на картинке видно"
- ❌ "в деталях указано"
- ❌ "как сказано в итоге"

Хорошие ответы:
- ✅ "О, это же Геральт! Какой угрюмый"
- ✅ "Ха, классический мем Distracted Boyfriend"
- ✅ "Выглядит как... Киану Ривз? Или похожий кто-то"

### 3. **Тест на узнавание**
Отправьте известного персонажа/мем:
- Бот должен назвать его по имени (Gemini узнал → DeepSeek использует)
- Если Gemini написал "похоже на X" → DeepSeek может сказать "кажется, это X"
- Если Gemini не узнал → DeepSeek не должен выдумывать имя

## Технические детали

### Как это работает сейчас

1. **Пользователь шлёт картинку** → `handlers.py` вызывает `describe_image_bytes()`
2. **Gemini** получает новый естественный промпт → возвращает описание как "рассказ"
3. **Описание добавляется в контекст:**
   ```
   [User: Вася] [Time: 14:30] [Message: смотри что нашёл] 
   [Image description: Мужик с длинными волосами, шрамы на лице, 
   два меча за спиной. Явно Геральт из The Witcher...]
   ```
4. **DeepSeek** видит системный промпт с VISION_PROMPT_SUFFIX → понимает, что описание = его "наблюдение"
5. **DeepSeek отвечает:** "О, Ведьмак! Геральт какой-то особо угрюмый на этом арте"

### Что видит DeepSeek в DEBUG режиме

При `DEBUG=True` в `llm_client.py` логируется:
```
[📤 ЗАПРОС К МОДЕЛИ]
[1] Role: SYSTEM
Content: You are Berangaria... 
=== IMAGES AND VIDEO ===
When a user sends media, you receive [Image description: ...]...

[2] Role: USER  
Content: [User: Вася] [Time: 14:30] [Message: смотри]
[Image description: Мужик с длинными волосами...]
```

Проверьте:
- Читается ли описание естественно?
- Нет ли в нём жёстких секций "1. ДЕТАЛИ"?

## FAQ

**Q: Почему описания всё равно на русском, а промпт на английском?**  
A: Это нормально. DeepSeek multilingual, легко переключается. Главное — чтобы описание звучало естественно на своём языке.

**Q: Может, перевести весь SYSTEM_PROMPT на русский?**  
A: Можно попробовать, но DeepSeek обучен на большем количестве английских инструкций — качество следования правилам может упасть. Лучше оставить base на английском, а русский контент инжектить как данные.

**Q: Бот всё равно иногда говорит "на картинке"?**  
A: Если редко (<10%) — это нормально, модель не идеальна. Если часто — усильте запрет в VISION_PROMPT_SUFFIX, добавьте больше примеров "что не надо говорить".

**Q: Можно ли учить бота на примерах хороших ответов?**  
A: Да! Добавьте в VISION_PROMPT_SUFFIX секцию с примерами:
```
Example good reactions:
User sends Geralt image → You: "О, Ведьмак! Откуда арт?"
User sends cat meme → You: "Котэ in a box. Классика жанра"
```
